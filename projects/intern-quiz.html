WellSky Intern Coding Challenge

WellSky | June 2026 | Audience: 2026 Intern Cohort
Python Testing TDD Technical Writing Curriculum Design

Overview

Designed and developed a hands-on coding challenge for the 2026 WellSky intern cohort, focusing on real-world refactoring skills, test-driven development, and professional workflow practices. The challenge provides interns with a realistic engineering scenario before working on production systems.

Purpose: Give new interns practical experience with refactoring messy code, writing tests, and following professional software development workflows using the same tools and processes used in production.

Challenge Design

The Scenario

Interns inherit a working but messy payroll processor and must refactor it—cleaning it up, making it testable, and fixing any bugs—without changing its behavior. This mirrors real-world scenarios where engineers must improve legacy code while maintaining functionality.

Core Requirements

  • Refactor code across five interconnected payroll processing files
  • Write comprehensive test suite (minimum 10 tests) covering all business rules
  • Follow Research → Plan → Implement → Validate workflow
  • Use WellSky Claude Code Plugins to structure work
  • Submit single PR with refactored code, tests, and detailed write-up

Required Workflow

1

Research

Understand existing code, identify issues, and document business rules before making any changes

2

Plan

Create detailed implementation plan outlining refactoring approach and test strategy

3

Implement

Write tests first (TDD), then refactor code while ensuring all tests pass

4

Validate

Verify behavior unchanged, all tests pass, and code quality improved

Technical Components

Payroll Processing System

The challenge includes five interconnected Python modules:

  • Employee Data: Employee information and tax withholding
  • Time Tracking: Hours worked, overtime calculations
  • Deductions: Benefits, retirement, taxes
  • Payment Processing: Net pay calculation and check generation
  • Main Orchestrator: Coordinates the entire payroll run

Testing Requirements

  • Minimum 10 tests covering all business rules
  • Tests must be written before refactoring (TDD approach)
  • Edge cases and error handling must be tested
  • All tests must pass before PR submission

Deliverable Structure

What Interns Submit

  • Code: Refactored payroll processor with improved structure
  • Tests: Comprehensive test suite (10+ tests)
  • PR Description: Detailed explanation of:
    • What issues were found in original code
    • What changes were made and why
    • How testing validates the refactor
    • Any trade-offs or design decisions

Impact & Design Philosophy

Real-World Preparation

The challenge intentionally mirrors real engineering work at WellSky: inheriting code, improving quality, and documenting decisions. It emphasizes that the write-up matters as much as the code—communication is crucial in professional settings.

Tool Familiarization

By requiring the WellSky Claude Code Plugins and structured workflow, interns gain hands-on experience with the same tools they'll use on production projects.

Key Learnings: Gained experience in technical curriculum design, creating realistic coding challenges, and structuring learning experiences that prepare interns for production engineering work. Learned to balance challenge difficulty with educational value.
