Data Formulator Architectural Refactor Review

WellSky | June 2026 | Author: Jaylen Sinclair | Reviewer: Venec Moy
Python FastAPI BigQuery Vertex AI Code Review Testing

Overview

Conducted comprehensive technical review of a massive architectural refactor that simplified the Data Formulator application by removing 10,804 net lines of code, locking to BigQuery-only data source, and integrating Vertex AI. The review identified critical breaking changes, security improvements, and test coverage enhancements.

Review Scope

This refactor represented a strategic pivot from a multi-source data platform to an enterprise-focused BigQuery-only solution. My review analyzed:

  • Architecture changes and breaking impacts
  • Test coverage and quality
  • Security improvements and risks
  • Migration path for existing users
  • Code quality and maintainability

✅ Strengths Identified

1. Exceptional Code Reduction

  • 10,804 lines removed through elimination of unused features
  • Deleted 12 Chinese translation files (~1,711 lines of i18n overhead)
  • Removed demo streaming backend (demo_stream.py - 1,168 lines)
  • Eliminated gallery UI (ChartGallery.tsx - 1,609 lines)
  • Removed 609-line translation guide and infrastructure

2. Strong Test Coverage

Identified 6 new comprehensive test suites:

  • test_bigquery_auto_provision.py (95 lines) - Auto-provisioning logic
  • test_bigquery_loader_caps.py (68 lines) - Loader limits
  • test_bigquery_only_registry.py (90 lines) - Registry lock enforcement
  • test_secure_filename.py (55 lines) - Security hardening
  • test_tool_show_user_data_preview.py (113 lines) - Preview tool testing
  • test_data_loaders_bigquery_only.py (69 lines) - API endpoint tests

Test Quality Assessment: Comprehensive edge case coverage, clear docstrings explaining rationale, proper use of pytest markers, clean mocking strategy.

3. WellSky Branding Integration

  • Updated index.html title and meta tags to "WellSky Data Management Portal"
  • Added WellSky logo (app-logo.png)
  • Removed external links and upstream project references
  • Consistent branding in App.tsx

4. Production-Ready BigQuery Improvements

  • Increased dataset enumeration from 50 to 1,000 max results
  • Increased table enumeration from 20 to 1,000 per dataset
  • Raised total table limit from 100 to 5,000 with warning logs
  • Implemented auto-provisioning via VERTEX_AI_PROJECT or BQ_PROJECT_ID
  • Default US location with BIGQUERY_LOCATION override

5. Vertex AI Integration

  • Added vertex_ai to BUILTIN_PROVIDERS
  • Authentication via ADC (Application Default Credentials) - no API keys needed
  • Project/location configuration via environment variables
  • Proper model registry integration

6. Security Improvements

  • Added filename sanitization testing
  • Removed demo data streaming endpoint (reduced attack surface)
  • Likely removed local folder loader (reduced arbitrary file access risk)

⚠️ Critical Concerns Identified

1. Breaking Changes Without Migration Path

Issue: Removal of all non-BigQuery data sources with no deprecation period or migration documentation.

Recommendation: Provide migration guide for users currently using PostgreSQL, MySQL, Snowflake, or local file loaders. Consider deprecation warnings before full removal.

2. Removed Functionality

Identified these removed features that may impact users:

  • PostgreSQL, MySQL, Snowflake data loaders
  • Local folder data loading
  • Chart gallery and demo datasets
  • Chinese language support

3. Documentation Gaps

Issue: No updated documentation reflecting BigQuery-only architecture.

Recommendation: Update README, user guides, and API docs to reflect architectural changes and new Vertex AI capabilities.
